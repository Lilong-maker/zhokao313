🔗 表关系说明

用户 → 订单 → 支付 → 退款
user.id → order_info.user_id → payment.user_id → refund.user_id
一个用户可以有多个订单，每个订单可以有多次支付记录，每次支付可以发起退款
活动 → 秒杀商品 → 库存
seckill_activity.id → seckill_product.activity_id → seckill_stock.seckill_product_id
一个活动包含多个秒杀商品，每个秒杀商品有独立的库存管理
商品 → 秒杀商品 → 订单商品
product.id → seckill_product.product_id → order_item.seckill_product_id
商品参与秒杀活动，订单商品记录购买详情
用户购买限制（限购控制）
UNIQUE KEY (user_id, activity_id, product_id)
唯一索引确保同一用户在同一活动中对同一商品只能购买限定次数
库存操作日志（审计追踪）
seckill_stock.id → stock_log.seckill_stock_id
记录所有库存变更操作，支持问题排查和数据恢复
